Git

  • ls: Displays a list of the files and directories of the directory you're in
  • pwd: "Print working directory", displays what directory you're in
  • cd directory-name: "Change directory", enters into a child directory.
    • cd .. : Exits into the parent directory
  • mkdir directory-name: Makes a new directory where you currently are
  • touch file-name.txt: Creates a new file
  • code . : Runs the current folder in Visual Studio
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
    • git checkout main: switches over to the main branch
  • git add -A: Commits all files in the working branch
  • git commit -m "added code to HTML file": Commits to the repository with an added message.

Saving Work

  1. Create new branch and verify branch
    • git checkout -b branch-name
    • git status
  2. Add changes wanted to the next commit
    • git add -A
  3. Commit the changes
    • git commit -m "commit notes here with parentheses"
  4. Pull main to ensure the code is up to date
    • git pull origin main
  5. Push the changes after the branch has been synced
    • git push origin branch-name
